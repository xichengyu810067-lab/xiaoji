小吉吉幣系統完整開發規格書
1. 專案目標

為 Discord 機器人「小吉」新增一套完整的虛擬貨幣系統，貨幣名稱暫定為 吉幣。

吉幣系統必須支援：

每日簽到取得吉幣
查詢自己的吉幣餘額
查詢其他人的吉幣餘額
吉幣排行榜
管理員加幣、扣幣、設定餘額
虛擬商店
使用吉幣兌換獎勵
背包或購買紀錄
重啟後資料不消失
GitHub 更新程式碼後不覆蓋資料庫
NyankoHost restart 後資料仍然保留

本系統只處理 Discord 內部的虛擬貨幣，不涉及真實金錢、加密貨幣、現金交易、賭博或任何外部支付。

2. 重要背景

小吉目前已經部署在 NyankoHost 上，並且已經有信任伺服器驗證系統。

目前已知小吉會在多個已批准伺服器中運作：

菇湯集團
★ Inifert ✧
社恐咖啡廳
☠NUKE-BRO-GG☠

請開發時務必保留既有功能，不可破壞：

現有 slash command
現有聊天功能
現有管理功能
現有信任伺服器 approved 機制
現有 .env 設定
現有 NyankoHost 部署方式
現有 GitHub 專案結構
3. 核心原則
3.1 GitHub 只放程式碼

GitHub repository 只能保存：

小吉的程式碼
指令邏輯
功能模組
設定範例
README 或文件

GitHub 不可以保存：

.env
Discord token
API key
正式吉幣資料庫
玩家吉幣資料
簽到紀錄
商店購買紀錄
信任伺服器資料
任何正式使用者資料
3.2 資料庫必須留在 NyankoHost

吉幣資料必須保存在 NyankoHost 的本地資料庫中。

更新程式碼時，不可以覆蓋或刪除正式資料庫。

NyankoHost restart 後，資料仍然必須存在。

3.3 小吉必須能自動建立資料庫

小吉啟動時要自動檢查資料庫是否存在。

如果資料庫不存在：

自動建立新的空資料庫
自動建立吉幣系統需要的資料結構
在 console 顯示資料庫建立成功

如果資料庫已經存在：

沿用既有資料
不可以重建
不可以清空
不可以覆蓋
在 console 顯示正在使用的資料庫路徑
3.4 資料保存必須安全

吉幣系統必須避免以下問題：

小吉重啟後玩家吉幣歸零
GitHub 更新後資料庫被覆蓋
同一時間多人操作導致資料錯亂
玩家重複簽到刷幣
非管理員亂加幣或扣幣
未批准伺服器使用吉幣系統
資料庫讀取失敗卻默默重建空資料庫
4. 資料庫規劃
4.1 建議使用 SQLite

請使用 SQLite 作為初期正式資料庫。

理由：

適合小型到中型 Discord bot
不需要額外資料庫伺服器
適合部署在 NyankoHost
方便備份
適合儲存吉幣、簽到、商店、背包、排行榜資料
4.2 資料庫存放位置

資料庫必須放在專門的資料資料夾中。

建議資料夾名稱：

data
或 database
或 storage

資料庫檔案名稱建議：

xiaoji.sqlite
或 xiaoji.db

請選擇一種固定命名方式，不要每次啟動產生不同檔案。

4.3 啟動時必須顯示資料庫資訊

小吉啟動時，console 必須顯示：

目前使用的資料庫路徑
資料庫是否已存在
是否有建立新資料庫
是否有建立新資料表
是否成功載入吉幣系統
若發生錯誤，必須清楚顯示錯誤原因
4.4 不可自動清空資料

任何情況下，程式都不可以在未經 owner 明確指令的情況下：

清空玩家資料
重置所有吉幣
刪除資料庫
覆蓋正式資料庫
把資料庫初始化成空資料

7. 預設經濟規則
7.1 每日簽到獎勵

預設規則：

每日簽到基本獎勵：50 吉幣
連續簽到第 3 天：額外 20 吉幣
連續簽到第 7 天：額外 100 吉幣
連續簽到中斷後，連續天數重置為 1

以上數值需要可在未來調整。

7.2 餘額限制

初版建議：

玩家最低餘額不可低於 0
玩家最高餘額暫不限制
管理員扣幣不可扣成負數
owner 可以有進階覆蓋權限，但必須明確記錄原因
7.3 轉帳功能

初版可以先不做轉帳。

但請保留未來擴充空間。

若要做轉帳，必須具備：

防止轉給機器人
防止轉給自己
防止負數或小數
防止短時間洗交易紀錄
轉出與轉入都要寫入交易紀錄
可由伺服器設定開關控制
8. 權限與安全要求
8.1 信任伺服器限制

吉幣系統只能在 approved 伺服器中使用。

如果伺服器不是 approved：

一般玩家不可使用吉幣系統
管理員不可使用吉幣系統
owner 可以看到警告與診斷資訊
不可自動替 unknown 伺服器建立完整經濟資料
8.2 管理員權限限制

以下功能不可給一般玩家使用：

加幣
扣幣

新增商品
編輯商品
停用商品

10. 防呆與錯誤處理
10.1 玩家不存在

如果玩家尚未有吉幣資料：

查詢時自動建立玩家資料
預設餘額為 0
不應報錯
10.2 餘額不足

購買商品時如果餘額不足：

不扣款
不給道具
回覆目前餘額與需要金額
10.3 商品不存在

如果商品 ID 不存在：

回覆商品不存在
不扣款
不寫入購買紀錄
10.4 商品已停用

如果商品已停用：

回覆商品目前不可購買
不扣款
10.5 庫存不足

如果商品庫存不足：

回覆庫存不足
不扣款
10.6 權限不足

如果使用者沒有權限：

回覆權限不足
不執行任何資料變更
不暴露內部錯誤細節
10.7 資料庫錯誤

如果資料庫讀寫失敗：

回覆「系統暫時無法處理吉幣資料」
console 顯示完整錯誤
不可以假裝成功
不可以重建空資料庫
不可以吞掉錯誤

13. 開發注意事項
13.1 不要破壞現有功能

請先閱讀現有專案結構，確認：

指令註冊方式
slash command 部署方式
權限判斷方式
owner ID 判斷方式
approved guild 判斷方式
NyankoHost 啟動方式
.env 使用方式

新增吉幣系統時，要盡量模組化，不要把所有功能塞進單一檔案。

13.2 不要覆蓋 .env

.env 是正式環境的重要設定檔，不可以覆蓋、刪除或推上 GitHub。

如果需要新增設定，請：

更新 .env.example
說明需要新增哪些環境變數
不要修改正式 .env 的既有敏感資料
13.3 不要把資料庫加入 GitHub

請確認 .gitignore 有排除：

.env
正式資料庫檔案
data 資料夾中的正式資料
database 資料夾中的正式資料
storage 資料夾中的正式資料
logs 中不適合公開的內容
13.4 更新程式碼不可刪除資料

如果專案有部署腳本或清理腳本，請確認它不會刪除：

data 資料夾
database 資料夾
storage 資料夾
正式 SQLite 資料庫
信任伺服器資料
