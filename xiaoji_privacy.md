## 1. 政策說明

本隱私權政策說明小吉在提供 Discord 機器人服務時，可能會收集、使用、保存、分享與刪除哪些資料。

當使用者將小吉加入 Discord 伺服器、使用小吉的指令、與小吉聊天、或讓小吉在伺服器內運作時，即表示使用者理解本政策所述的資料處理方式。

若使用者不同意本政策，請停止使用小吉，並可將小吉從 Discord 伺服器中移除。

---

## 2. 資料處理原則

小吉處理資料時，遵循以下原則：

1. **必要性原則**：只在提供功能、維護服務、防止濫用或除錯所需範圍內處理資料。
2. **最小化原則**：不主動要求與功能無關的個人資料。
3. **透明原則**：清楚說明可能收集的資料類型與用途。
4. **安全原則**：盡力以合理方式保護資料安全。
5. **可刪除原則**：使用者或伺服器管理員可依本政策請求刪除相關資料。

若小吉的實際程式版本沒有啟用某項功能，則該功能相關資料不會被小吉主動收集或保存。

---

## 3. 小吉可能收集的資料

小吉可能依功能需求收集以下資料：

### 3.1 Discord 基本識別資料

小吉可能收集：

- 使用者 ID
- 使用者名稱
- 使用者顯示名稱
- 伺服器 ID
- 伺服器名稱
- 頻道 ID
- 頻道名稱
- 身分組 ID
- 身分組名稱
- 訊息 ID
- 指令互動 ID
- 時間戳記

這些資料主要用於識別誰使用了功能、在哪個伺服器或頻道使用功能，以及如何正確執行指令。

---

### 3.2 指令與互動資料

當使用者執行小吉指令或與小吉互動時，小吉可能收集：

- 指令名稱
- 指令參數
- 指令執行時間
- 指令執行結果
- 指令錯誤紀錄
- 冷卻時間與使用次數
- 是否具有使用該指令的權限

這些資料用於執行功能、排除錯誤、防止濫用、限制 API 額度，以及改善服務品質。

---

### 3.3 訊息內容

小吉可能在以下情況讀取或處理訊息內容：

- 使用者提及小吉並與小吉聊天
- 使用者使用文字指令
- 使用者傳送需要小吉處理的內容
- 使用者要求 AI 回覆
- 小吉需要偵測洗頻、炸群、濫用或違規行為
- 小吉需要執行公告、管理、遊戲或其他互動功能

小吉不會主動要求使用者提供密碼、Discord Token、API Key、信用卡資料、銀行資料或其他高度敏感資訊。使用者也不應將這些資訊傳送給小吉。

---

### 3.4 AI 聊天資料

若使用者使用小吉的 AI 聊天功能，小吉可能處理：

- 使用者傳送給小吉的訊息內容
- 小吉產生的回覆內容
- 對話上下文
- 使用者 ID、伺服器 ID、頻道 ID
- 使用量、限制次數或錯誤紀錄

這些資料可能會被傳送至第三方 AI 服務，以產生回覆或完成請求。

使用者不應在 AI 聊天中輸入密碼、Token、金流資料、身分證件、私人住址、未公開個人資料或其他敏感資訊。

---

### 3.5 伺服器管理與安全資料

若小吉提供管理、防洗頻、防炸群、公告或審核功能，可能收集：

- 管理指令使用紀錄
- 被管理對象的使用者 ID
- 操作者的使用者 ID
- 踢出、停權、禁言、刪除訊息或身分組異動紀錄
- 伺服器核准狀態
- 白名單或黑名單資料
- 邀請審核資料
- 異常行為紀錄
- 防洗頻、防炸群偵測資料

這些資料用於協助伺服器管理員維持伺服器秩序、防止濫用，以及追蹤重要管理操作。

---

### 3.6 虛擬經濟與遊戲資料

若小吉提供吉幣、簽到、工作、銀行、商店、道具、對戰、排行榜或小遊戲功能，可能收集：

- 使用者 ID
- 吉幣餘額
- 銀行存款資料
- 利息計算資料
- 工作職業、工作天數、工作開始與結束時間
- 簽到紀錄
- 商店購買紀錄
- 虛擬道具資料
- 身分組或頻道兌換紀錄
- 對戰紀錄
- 虛擬 HP、技能、牌位、分數或排行榜資料

這些資料僅用於小吉內部的虛擬娛樂與功能互動，不具有現實金錢價值。

---

### 3.7 天氣、音樂與外部查詢資料

若使用者使用天氣、音樂或其他外部查詢功能，小吉可能處理：

- 使用者輸入的地點關鍵字
- 查詢時間
- 查詢結果
- 音樂或影片連結
- 歌曲標題、播放清單或音訊狀態
- 語音頻道 ID
- 相關錯誤紀錄

這些資料用於完成查詢、播放音訊、處理佇列或排除錯誤。

---

### 3.8 技術與除錯資料

為維持服務穩定，小吉可能收集：

- 錯誤訊息
- 系統日誌
- API 回應狀態
- 資料庫連線狀態
- 指令失敗紀錄
- 執行環境資訊
- 效能資料
- 服務啟動、關閉或重啟紀錄

這些資料用於維護、除錯、改善效能與防止服務濫用。

## 4. 小吉不會主動收集的資料

小吉不會主動要求或刻意收集以下資料：

- Discord 密碼
- Discord Token
- API Key
- 信用卡號
- 銀行帳號
- 身分證件影本
- 真實住址
- 真實金流資料
- 與小吉功能無關的高度敏感個人資料

若使用者主動把上述資料傳送到 Discord 頻道或傳給小吉，使用者應自行承擔相關風險。開發者建議使用者立即刪除相關訊息，並視情況更換密碼、重設 Token 或採取安全措施。

---

## 5. 資料使用目的

小吉可能將資料用於以下目的：

1. 執行使用者請求的指令。
2. 提供 AI 聊天、天氣、音樂、公告、管理、遊戲與虛擬經濟功能。
3. 保存吉幣、銀行、工作、商店、對戰、排行榜或其他遊戲進度。
4. 判斷使用者是否具有使用特定指令的權限。
5. 執行冷卻時間、每日限制、伺服器限制或 API 額度控管。
6. 防止洗頻、炸群、惡意操作、漏洞濫用或未授權使用。
7. 記錄重要管理操作，協助伺服器管理員維護秩序。
8. 排除錯誤、改善穩定性與提升服務品質。
9. 遵守 Discord 政策、平台要求、法律義務或安全需求。

## 6. 第三方服務與資料分享

小吉可能為了提供功能，將必要資料傳送至第三方服務。可能涉及的第三方服務包括：

- Discord API
- AI API 或大型語言模型服務
- 天氣資料 API
- 音樂、影片或音訊相關服務
- 主機服務
- 資料庫服務
- 日誌、監控或錯誤追蹤服務

小吉只會在完成使用者請求、提供功能、維持服務或除錯所需範圍內分享必要資料。

開發者不會出售 Discord API 資料，不會將使用者資料作為獨立商品販售，也不會將小吉收集的資料用於與小吉功能無關的商業轉售。

第三方服務可能依其自身服務條款與隱私權政策處理資料。使用者在使用相關功能時，也應理解第三方服務可能有自己的資料處理方式。

## 7. 資料保存期間

小吉會依照功能需要保存資料。不同資料的保存期間可能不同：

- **伺服器設定資料**：通常保存至伺服器移除小吉、設定被重置，或管理員請求刪除為止。
- **吉幣、銀行、工作、商店與遊戲資料**：通常保存至功能停用、資料重置、使用者請求刪除，或開發者因維護需要清理為止。
- **管理與安全紀錄**：可能保存一段時間，以供伺服器管理、濫用調查或安全追蹤使用。
- **AI 聊天與訊息處理資料**：可能在完成功能後不再長期保存；若系統需要上下文、除錯或濫用防治，可能在必要範圍內保存。
- **錯誤與技術日誌**：可能保存一段時間，以供維護、除錯與改善服務使用。

若資料已無保存必要，開發者可刪除、匿名化、覆寫或重置相關資料。

## 8. 資料刪除請求

使用者可以聯絡開發者，請求刪除與自己相關的小吉資料。

伺服器擁有者或具有適當權限的伺服器管理員，也可以請求刪除該伺服器與小吉相關的伺服器設定資料、經濟資料、遊戲資料或管理資料。

提出刪除請求時，請盡量提供以下資訊，以利確認資料範圍：

- Discord 使用者 ID
- Discord 伺服器 ID
- 希望刪除的資料類型
- 請求原因或補充說明

開發者會在合理可行範圍內處理刪除請求。但在以下情況，部分資料可能無法立即刪除：

1. 需要保留以防止濫用、詐騙、攻擊或違規行為。
2. 需要保留以排除錯誤或維持服務安全。
3. 需要保留以遵守法律、平台政策或合理管理需求。
4. 資料已被備份、匿名化、彙整化，或無法再直接對應到特定使用者。

## 9. 資料安全

開發者會盡力以合理方式保護小吉相關資料，例如：

- 限制資料存取權限
- 避免公開 Token、API Key 或敏感設定
- 使用環境變數或安全方式保存機密資料
- 依需求維護主機、資料庫與程式
- 修正已知錯誤與安全問題

然而，任何網路服務都無法保證百分之百安全。使用者理解並同意，資料傳輸、第三方服務、主機環境、Discord 平台或使用者自身帳號安全，都可能存在風險。

## 10. 兒童與未成年使用者

小吉運作於 Discord 平台上。使用者應符合 Discord 對帳號年齡與使用資格的要求。

若未成年使用者使用小吉，應在家長、監護人或伺服器管理員的適當監督下使用。小吉不會刻意針對兒童收集與功能無關的個人資料。

## 11. 國際資料傳輸

小吉使用的 Discord、主機、AI API、天氣 API、資料庫或其他第三方服務，可能位於使用者所在地以外的地區。

當使用者使用小吉時，相關資料可能會依功能需求被傳輸、處理或保存於不同地區的伺服器。使用者應理解不同地區可能有不同的資料保護法律與規範。

---

## 12. 伺服器管理員的責任

將小吉加入伺服器的擁有者或管理員，應負責確認：

1. 伺服器成員知道小吉可能會處理與指令、訊息、管理、遊戲或經濟功能相關的資料。
2. 小吉被授予的權限符合伺服器實際需求。
3. 管理指令、公告指令、經濟指令或其他敏感功能只開放給適當人員使用。
4. 若伺服器有自己的隱私規則或公告，應與本政策保持一致或採取更嚴格的保護。
